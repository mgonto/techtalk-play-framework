¿Que acabamos de ver?

Controllers

  • Los métodos son públicos y estáticos
  • Los redirects se hacen llamando a otro método
  • El binding de los objetos se realiza automaticamente
  • Reconoce solo los nombres de las variables a enviar al template
  • Render y RenderArgs son usados para enviar objetos a la vista
  • Se utilizan validations y params para bindear y validar
  • JSON made easy con renderJson

Vistas

  • Las vistas utilizan el lenguaje de scripting Groovy
  • Los helpers que tiene son muy similares a los de JSP
  • La herencia de templates se declara inline.
  • La creacion de URLs absolutas y relativas ya esta resuelto

Modelos

  • Los modelos se crean con anotaciones normal de JPA
  • Cuentan con métodos que facilitan el ABMC de los mismos
  • Las propiedades deben ser públicas
  • Los getters y setters son creados automáticamente
  • Las validaciones declaradas aqui son usadas tanto en la vista como en la interacción con la DB
  • El comportamiento debe estar en los modelos, eliminando asi la necesidad de IoC

Rutas

  • Se encuentran todas las rutas en un solo archivo
  • Es mucho mas fácil "tracearlas"
  • Permite crear rutas con Regex y parametros

Let's deploy this app!

Dale Play!

Flash, Params y Validaciones

  • Play! cree que las WebApps deben ser stateless
  • Desalienta el uso de la session
  • El flash es una cookie que vive un request
  • El flash es utilizado para validaciones, mensajes y parámetros HTTP
  • La cache es utilizada para guardar objetos que deben permanecer a lo largo de los requests

Tests

  • Permite crear tests unitarios a los cuales estamos acostumbrados
  • Permite crear tests funcionales que levantan un server web inline sin necesidad de un browser
  • Permite crear tests de Selenium para probar el flujo y la UI
  • Provee una herramienta web para correrlos facilmente

Magics explained

Controllers Magic

  • Usa ByteCode Enhancement para ver que variables fueron llamadas en un metodo y saber los nombres para mandarlos al template
  • Tiene un objeto singleton que le permite determinar si se esta dentro de una accion. Si es asi, llamar a otra accion hace un redirect

Models Magic

  • Cada vez que se llama a una property de un objeto, llama al getter o setter de la misma
  • Si este getter ya existia, no lo crea. En caso contrario lo crea.
  • Para todo esto utiliza ByteCode Enhancement

Módulos

Que es un módulo?

  • Es una dependencia del proyecto muy similar a un JAR
  • Permite crear dependencias no solo de código Java, sino de HTML y JS de forma muy facil
  • Hay una gran comunidad que crea módulos para Play!

Módulos Conocidos

  • Secure

    • Es un módulo que nos permite agregarle login a nuestra app de forma muy simple

  • Greenscript

    • Es un módulo que nos permite en producción minimizar, juntar y comprimir los JS y CSS
    • Nos permite crear dependencias entre los JS y CSS

  • CRUD

    • Nos permite crear Scaffolds para las aplicaciones
